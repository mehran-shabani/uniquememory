# بک‌لاگ اجرایی اولیه (MVP)

## Epic 1 — راه‌اندازی زیرساخت پایه
- [ ] ایجاد پروژهٔ Django + DRF، پیکربندی settings محلی.
- [ ] پیاده‌سازی OAuth2/OIDC Proxy ساده با JWT (Stub یا Auth0 داخلی).
- [ ] تعریف مدل‌های users، companies، agents، clients.
- [ ] راه‌اندازی pipeline CI (lint + تست واحد).

## Epic 2 — هستهٔ حافظه و رضایت
- [ ] مدل‌های memory_entries، entry_chunks، consents، audit_logs.
- [ ] APIهای CRUD با کنترل نسخه و سیاست دسترسی.
- [ ] مدیریت رضایت (ایجاد، لغو، تاریخچه) + اتصال به ماتریس حساسیت.
- [ ] لاگ ممیزی با فیلتر براساس scope و حساسیت.

## Epic 3 — جست‌وجوی هیبریدی و کش
- [ ] سرویس Chunking و Embedding (Stub اولیه با مدل محلی).
- [ ] ساخت ایندکس برداری فایل‌محور و جست‌وجوی ترکیبی (BM25 + Cosine).
- [ ] پیاده‌سازی کش محلی برای نتایج جست‌وجو با TTL قابل تنظیم.
- [ ] تست پایداری revoke روی کش (invalidate < ۳۰ ثانیه).

## Epic 4 — فایل‌سیستم و پیوست‌ها
- [ ] ماژول آپلود پیوست با مسیر هش‌شده و کنترل نوع فایل.
- [ ] تعریف Job بکاپ روزانه و بازیابی دستی.
- [ ] مستندسازی سیاست نگهداری و حذف امن.

## Epic 5 — پرتال و SDK پایه
- [ ] فعال‌سازی Django Admin برای مدیریت اولیهٔ داده.
- [ ] ساخت صفحهٔ ساده برای مدیریت رضایت کاربر (template Django).
- [ ] تهیهٔ نمونهٔ SDK Python برای دسترسی به API.
- [ ] مستندسازی سناریوهای استفاده در Postman Collection.

## Epic 6 — امنیت و مشاهده‌پذیری
- [ ] پیکربندی TLS معکوس (Nginx یا Caddy) و لاگ سرور.
- [ ] راه‌اندازی SAST و Dependency check در CI.
- [ ] تنظیم alert روی رویدادهای حساس (دسترسی high/critical).
- [ ] نوشتن Runbook واکنش به رخداد امنیتی.

## Epic 7 — اقدامات اصلاحی پایلوت
- [ ] جداسازی صف Query از فرآیند Embedding با استقرار Redis اختصاصی برای کاهش Latency جست‌وجوی تراکنش.
- [ ] بازتنظیم وزن فیلدهای KYC و بازآموزی مدل رتبه‌بندی برای افزایش دقت به بالای 92%.
- [ ] غنی‌سازی داده‌های رهگیری با فیلد ETA مصنوعی و تست صحت دقت سناریوی مسیر حمل.
- [ ] خودکارسازی pipeline بارگذاری داده شرکا با اسکریپت قابل زمان‌بندی.

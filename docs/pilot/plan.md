# برنامه پایلوت

## اهداف

- اعتبارسنجی عملکرد سامانه حافظه منحصربه‌فرد در سناریوهای واقعی B2B.
- ارزیابی آماده‌سازی داده و یکپارچگی با سیستم‌های شریک.
- اندازه‌گیری KPIهای کلیدی: **Latency** (میانگین زمان پاسخ API) و **دقت** (نسبت پاسخ‌های صحیح).

## شرکت‌های منتخب و داده‌های آزمایشی

| شرکت | صنعت | سناریوی داده | حجم رکورد | توضیحات آماده‌سازی |
|------|-------|--------------|------------|---------------------|
| آلفا تک | فین‌تک | تراکنش‌های کارتی شبه‌واقعی | 250k | داده‌ها با استفاده از مولد synthetic و اعمال قوانین تطبیق KYC آماده شد. |
| سلامت نو | سلامت دیجیتال | پرونده‌های پزشکی ناشناس‌سازی‌شده | 80k | داده‌ها مطابق HIPAA با ماسک‌گذاری شناسه‌ها و استاندارد HL7 بارگذاری شد. |
| لجستیک پلاس | لجستیک | سفارش‌های حمل و نقل بین‌المللی | 120k | داده‌ها از ERP نمونه استخراج و با شناسه‌های رهگیری تصادفی غنی‌سازی گردید. |

## سناریوهای پایلوت

1. **جست‌وجوی حافظه تراکنش‌های مالی**
   - هدف: بهبود بازیابی پرداخت‌های مشکوک برای تیم‌های ضدتقلب.
   - جریان: API جست‌وجو → رتبه‌بندی حافظه → ارائه پاسخ قابل توضیح.
2. **پیشنهاد درمان‌های مشابه**
   - هدف: سرعت‌بخشی به توصیه درمانی با استفاده از حافظه پرونده‌های مشابه.
   - جریان: ingestion روزانه → Embedding حافظه → پاسخ درمانی.
3. **بهینه‌سازی مسیر حمل**
   - هدف: کاهش زمان پاسخ تیم پشتیبانی به سوالات رهگیری.
   - جریان: وب‌هوک سفارش → ذخیره‌سازی حافظه → API رهگیری.

## طراحی آزمایش و KPI

| سناریو | Latency هدف | Latency اندازه‌گیری | دقت هدف | دقت اندازه‌گیری | وضعیت |
|--------|--------------|----------------------|---------|------------------|--------|
| جست‌وجوی تراکنش | ≤ 800ms | 920ms | ≥ 92% | 89% | نیاز به بهبود |
| پیشنهاد درمان | ≤ 1.2s | 980ms | ≥ 90% | 93% | موفق |
| مسیر حمل | ≤ 700ms | 650ms | ≥ 88% | 85% | نیاز به بهبود دقت |
